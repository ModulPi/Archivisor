## 一、项目概述

### 1.1 项目定位

**个人数据治理 Agent** 是一款 AI 驱动的桌面应用，旨在帮助用户从"被动管理文件"转变为"主动治理数据"。它是一位 **"个人数据架构师"**——能够理解用户的工作习惯，设计合理的存储结构，并持续维护数据的有序流转。

### 1.2 核心理念

> **用户不感知文件管理，但文件始终有序。**

| 传统工具 | 本产品 |
|----------|--------|
| 用户主动操作，工具被动响应 | Agent 主动感知，智能化治理 |
| 单次清理，治标不治本 | 持续维护，形成治理闭环 |
| 统一规则，无法个性化 | 基于用户画像，量身定制 |
| 工具视角（文件/文件夹） | 架构师视角（结构/流转/生命周期） |

### 1.3 目标用户

| 用户类型 | 典型场景 | 核心痛点 |
|----------|----------|----------|
| **开发者** | 多项目并行，环境复杂 | 项目文件散落各处，依赖管理混乱，难以快速定位 |
| **创意工作者** | 大量素材、工程文件 | 素材占用巨大，版本管理困难，归档无章法 |
| **普通用户** | 日常办公、娱乐、生活 | 下载文件夹杂乱，桌面堆积，磁盘频繁告急 |
| **混合型用户** | 上午编程，下午剪辑，晚上游戏 | 多身份切换时，治理策略无法随场景调整 |


## 二、用户画像与核心场景

### 2.1 四类典型用户画像

| 画像 | 职业/身份 | 典型工作流 | 核心诉求 |
|------|----------|------------|----------|
| **Alex（开发者）** | 全栈工程师 | 克隆项目→配置环境→编码→测试→提交→归档 | 项目结构标准化、依赖隔离、快速切换上下文 |
| **Sophia（创意工作者）** | 视频剪辑师/设计师 | 收集素材→粗剪→精剪→导出→归档 | 素材自动归类、大文件管理、版本追溯 |
| **Mike（混合型用户）** | 自由职业者 | 上午编程、下午设计、晚上学习 | 多身份治理策略自适应，互不干扰 |
| **Emma（普通用户）** | 办公白领 | 日常办公文档、个人照片、娱乐资源 | 桌面整洁、下载自动分类、清理建议简单明了 |

### 2.2 核心业务场景

**场景一：新设备/新用户初始化**

Emma 换了新电脑，希望快速建立一套适合自己的文件管理体系。Agent 通过 3-5 个问题了解她的职业和使用习惯，自动生成一套推荐的文件夹结构方案，Emma 确认后一键执行。

**场景二：项目启动（脚手架）**

Alex 开始一个新 Python 项目，Agent 根据他之前的项目模板，在 `Projects/Dev/` 下自动创建标准化的项目结构（含 `src/`、`tests/`、`docs/`、`requirements/`），并建议依赖存放位置。

**场景三：日常数据流转**

Mike 下载了一个素材包，Agent 识别到是视频素材，根据当前活跃上下文（他上午在剪辑模式）自动归类到 `Media/Projects/Current/`。截图自动存入 `Workspace/Screenshots/`，超过 30 天的临时文件自动提醒清理。

**场景四：项目完结归档**

Alex 完成了一个项目，告诉 Agent"这个项目做完了"。Agent 询问是归档还是删除，确认后执行：打包压缩→移动到 `Archive/Projects/`→更新索引→记录审计日志。

**场景五：数据检索**

Sophia 想找"上次剪辑用的那段配音"，Agent 通过语义检索（第二阶段）定位到 `Archive/Projects/2025/Q4/Project_Alpha/Audio/voiceover_final.wav`。

**场景六：存储空间预警**

磁盘使用率超过 85%，Agent 主动生成报告：列出可清理的大文件、重复文件、过期缓存，并给出"释放 50GB 空间"的一键方案。

**场景七：身份切换**

Mike 结束编程任务，告诉 Agent"切换到设计模式"。Agent 调整治理策略：后续下载的素材归类到 `Media/` 而非 `Projects/Dev/`，同时加载设计相关的项目模板。

**场景八：即时通讯文件统一管理**

Bob 日常使用微信、QQ 等软件进行工作沟通，截图、音频、视频、文档散落在各自默认目录中。Agent 自动接管：截图统一收纳、接收文件按类型自动归类、音频视频集中存储，并每月定期清理提醒。

**场景九：录屏/直播内容自动归位**

Tom 在游戏时开启录屏工具录制精彩片段，录屏软件将视频存入默认目录。Agent 监控该目录，一旦检测到新视频立即按预设结构迁移到 `Media/Gaming/Captures/`，按游戏名和日期自动创建子文件夹，并联动命名规范。

**场景十：跨设备文件同步与冲突处理（远期展望）**

Lisa 在公司台式机和家庭笔记本之间来回工作，U 盘频繁插拔。Agent 自动识别移动设备，对比文件版本差异，检测冲突并自动创建两个版本并存，支持一键同步/备份。

> **注**：本场景涉及设备管理和文件同步能力，当前版本核心功能中作为远期规划，不纳入 MVP 范围。

**场景十一：软件安装自动数据重定向**

Bob 新安装了一款软件（如微信、游戏、开发工具），Agent 自动感知到新软件的存在，识别其数据目录位置，通过符号链接技术将数据透明地重定向到预先设计好的治理结构中。软件下次启动时，完全无感知地读写新位置的数据。


## 三、功能需求

### 3.1 第一阶段：基础层（MVP）

**目标**：搭建项目骨架，实现数据采集、基础操作和可视化看板。

#### 子阶段 1a（第 1-2 周）—— 核心数据底座

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **数据采集层** | 文件元数据索引 | 扫描指定目录，记录文件名、路径、大小、扩展名、时间戳、文件 Hash |
| | 操作日志记录 | 每次文件操作自动写入审计日志，含操作类型、时间、路径 |
| | 目录监听 | 监控指定目录变化，实时更新索引 |
| **文件操作** | 查询/迁移/复制/删除/备份 | 支持按多条件查询，批量操作，删除优先入回收站 |
| **数据看板** | 磁盘使用概览 | 环形图展示各分区使用率 |
| | 文件类型分布 | 按扩展名分类展示数量与大小占比 |
| | 大文件 Top N | 按大小排序展示文件列表 |
| | 最近变更 | 展示最近新增/修改/删除的文件 |
| | 重复文件检测 | 基于 Hash 检测并展示重复文件 |
| **系统集成** | 符号链接管理 | 创建/删除/查询目录符号链接 |
| | 迁移执行与回滚 | 执行数据迁移，支持撤销操作 |

#### 子阶段 1b（第 3-4 周）—— 交互层与事件骨架

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **用户画像** | 冷启动问卷 | 首次使用时 5-8 个问题建立初始画像 |
| | 画像存储与编辑 | 支持查看和修改画像信息 |
| **对话窗口** | 基础聊天界面 | 支持 Markdown 渲染，自然语言交互 |
| | 流式输出 | 支持打字机效果展示回复 |
| **系统集成** | 软件安装感知 | 监控 Program Files、AppData 等目录变化 |
| | 软件类型识别 | 根据软件名称和路径特征识别类型 |
| | 数据目录发现 | 识别软件的默认数据存储路径 |
| **事件驱动** | 文件系统事件监听 | 监听新增/修改/删除事件，自动触发索引更新 |
| | 系统监控事件 | 监听磁盘空间变化，触发预警 |
| **可观测性** | 结构化日志 | JSON 格式日志，便于分析 |
| | 关键指标采集 | 记录响应时间、成功率等 |

### 3.2 第二阶段：智能层

**目标**：Agent 具备分析和设计能力，能理解磁盘现状、理解用户意图，输出个性化存储架构方案。

> **画像持续积累**：第二阶段运行时，用户每次确认/拒绝 Agent 方案，画像系统同步记录这些偏好（如"用户拒绝了某方案""用户接受了某分类规则"），作为后续持续学习的基础数据。

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **Agent核心** | 意图识别 | 识别用户指令类型（查询/执行/设计/规则/回溯） |
| | 任务规划 | 基于意图拆解任务，制定工具调用序列 |
| | 工具调用 | 按规划调用文件系统/分析/治理工具 |
| | 状态管理 | 维护会话状态和任务执行状态 |
| | 记忆系统 | 短期记忆（会话上下文）+ 长期记忆（画像+历史） |
| **分析能力** | 磁盘使用分析 | 分析使用率、分布趋势 |
| | 文件分类分析 | 多维度分析文件类型/大小/时间/访问频率 |
| | 垃圾文件识别 | 识别临时、缓存、重复、过期文件 |
| | 使用习惯分析 | 分析高频访问文件夹、文件类型偏好 |
| **设计能力** | 架构方案生成 | 基于画像+磁盘现状+用户描述生成个性化方案 |
| | 方案对比 | 提供多套方案对比 |
| | 方案预览与确认 | 树形结构展示，用户确认后执行 |
| **脚手架能力** | 项目结构生成 | 根据项目类型自动生成标准化结构 |
| | 模板管理 | 用户可自定义/修改项目模板 |

### 3.3 第三阶段：自动化层

**目标**：Agent 从"被动响应"升级为"主动治理"，实现自动化、规则化的数据流转。

| 模块 | 功能点 | 说明 |
|------|--------|------|
| **Skill引擎** | Skill 定义与管理 | 将复合工作流封装为可复用的 Skill |
| | Skill 匹配与调度 | 根据用户指令匹配最合适的 Skill |
| | Skill 自动沉淀 | 成功执行的未知场景经优化后生成 Skill 草稿 |
| | Skill 热度统计 | 记录使用次数和成功率 |
| **规则引擎** | 规则定义 | 用户自定义条件→动作规则 |
| | 规则模板 | 提供常用规则模板 |
| **自动路由** | 下载自动归档 | 监控 Downloads，按规则自动归类 |
| | 截图自动归档 | 监控 Screenshots，按日期/内容归类 |
| | 临时文件管理 | 过期文件自动清理提醒 |
| **定时治理** | 定期扫描/清理 | 按设定频率自动执行 |
| | 空间预警 | 超阈值时主动推送预警 |
| **画像学习** | 行为学习 | 从审计日志学习用户偏好 |
| | 主动建议 | Agent 主动提出优化建议 |
| **审计与恢复** | 操作审计 | 完整记录所有 Agent 操作 |
| | 一键回滚 | 支持撤销最近操作 |
| **冷热数据分离** | 元数据归档 | 超期数据自动迁移到冷存储 |
| | 审计日志分区 | 按季度自动分表 |
| **Skill 预加载** | 启动时预热 | 高频 Skill 启动时预加载到内存 |
| | 场景感知预判 | 根据用户当前行为预加载相关 Skill |
